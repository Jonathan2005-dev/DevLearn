Lesson 11: HTML Best Practices

Code organization, naming conventions, and standards

📚 Learning Objectives

  • ✓ Follow HTML naming conventions
  • ✓ Structure code for readability
  • ✓ Comment code effectively
  • ✓ Validate HTML code
  • ✓ Follow web standards

✅ Do's

  • ✓
    Use semantic HTML

    Use meaningful tags like header, nav, main, article

  • ✓
    Indent code properly

    Use consistent spacing (2 or 4 spaces)

  • ✓
    Use lowercase tag names

    <div> not <DIV>

  • ✓
    Quote attribute values

    href="page.html" not href=page.html

  • ✓
    Close all tags

    Include closing tags for all elements

❌ Don'ts

  • ✗
    Avoid div-itis

    Don't use divs for everything

  • ✗
    Don't ignore accessibility

    Always add alt text, use labels

  • ✗
    Don't nest improperly

    Maintain proper HTML hierarchy

  • ✗
    Don't use inline styles

    Use external CSS files instead

  • ✗
    Don't mix concerns

    Keep HTML, CSS, JS separate

📝 Code Organization Example

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width">
    <title>Page Title</title>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <header>
        <h1>Site Title</h1>
    </header>

    <main>
        <article>
            <h2>Article Title</h2>
            <p>Article content...</p>
        </article>
    </main>

    <footer>
        <p>© 2024 My Site</p>
    </footer>

    <script src="script.js"></script>
</body>
</html>
